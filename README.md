# Docker-проект: контейнеризация приложения

## О проекте

Это курсовая работа по Docker. Приложение состоит из двух частей:
- **Бэкенд** на Go (простое API с healthcheck)
- **Фронтенд** на Vue.js (сборка через nginx)

Всё завернуто в контейнеры, запускается через docker-compose. Постарался сделать безопасно и оптимизированно.

## Что внутри

### Бэкенд
- Многоэтапная сборка: сначала собираем бинарник в `golang:alpine`, потом копируем в чистый `alpine`
- Бинарник весит ~15 МБ (вместе с alpine ~20 МБ)
- Запускается от своего пользователя `appuser`, не от root
- Есть healthcheck (проверяет `/health`)
- Сканировал через `trivy` — критичных уязвимостей нет

### Фронтенд
- Тоже multi-stage: собираем в `node:alpine`, потом копируем собранный `dist` в `nginx:alpine`
- Nginx настроен на отдачу статики, запускается от пользователя `nginx`
- Размер образа ~25 МБ (из-за nginx)
- Пришлось добавить `NODE_OPTIONS=--openssl-legacy-provider`, потому что старый webpack не дружит с новым OpenSSL

### Docker Compose
Поднимаются два сервиса:
- `backend` на порту `8081`
- `frontend` на порту `80` (если порт занят, можно поменять в docker-compose.yml)

Сети изолированные, контейнеры общаются через внутреннюю сеть. У бэкенда настроен healthcheck и ограничения по CPU/памяти.

## Как запустить

### Локально

```bash
# Клонируем
git clone <ваш репозиторий>
cd cloud-services-engineer-docker-project-sem2

# Запускаем
docker-compose up -d

# Проверяем
docker-compose ps
curl http://localhost:8081/health   # бэкенд
curl http://localhost/              # фронтенд (или localhost:80)
